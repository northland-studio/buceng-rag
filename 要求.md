## 项目书

### 1. 项目目标
- 研究人员在 Web 界面输入游戏内结构化的事件文本（例如：“玩家在公共农场收割后不补种，导致冲突并催生规则”）。
- 系统自动从本地向量知识库中检索最相关的社科理论卡片（例如“公地悲剧”、“社会契约”）。
- 将检索到的理论与事件一起发送给云端大模型（DeepSeek API），生成带理论引用的分析报告。
- 界面显示分析结果、引用理论原文，并允许研究人员对结果评分并保存为“黄金样本”（用于未来微调）。
- 整个知识库和嵌入模型完全在本地运行，保证理论卡片和数据不外泄。

### 2. 技术栈要求
- Python 3.10+
- Web 框架：Streamlit
- 向量数据库：ChromaDB（本地模式，数据存储在本地文件夹 `./chroma_db`）
- 嵌入模型：`BAAI/bge-large-zh-v1.5`，使用 `sentence-transformers` 加载，运行在本地
- LLM：通过 OpenAI 兼容接口调用 DeepSeek API（`deepseek-chat`），使用 `openai` Python 库
- 配置管理：使用 `.env` 文件存储 API 密钥，通过 `python-dotenv` 加载

### 4. 各模块详细要求

#### config.py
- 使用 `dotenv` 加载 `.env` 文件。
- 读取：`DEEPSEEK_API_KEY`，`DEEPSEEK_BASE_URL`（默认 `https://api.deepseek.com`），`EMBEDDING_MODEL_NAME`（默认 `BAAI/bge-large-zh-v1.5`），`CHROMA_PERSIST_DIR`（默认 `./chroma_db`），`COLLECTION_NAME`（默认 `sociology_cards`）。

#### embedding.py
- 使用 `sentence_transformers.SentenceTransformer` 加载 `config.EMBEDDING_MODEL_NAME`。
- 提供一个函数 `get_embedding(text: str) -> list[float]`，返回向量。
- 注意：模型首次运行时会自动下载，请设置国内镜像（如有需要可提示用户手动下载）。

#### knowledge_base.py
- 初始化 ChromaDB 客户端（`PersistentClient`），路径来自 `config.CHROMA_PERSIST_DIR`。
- 获取或创建集合 `COLLECTION_NAME`，并指定嵌入函数为自定义的 `embedding_fn`（接受文本列表返回向量列表）。**Chroma 允许传入 `embedding_function`**，请实现一个包装类 `LocalEmbeddingFunction`，内部调用 `embedding.get_embedding`。
- 函数 `add_cards(cards: list[dict])`：每个卡片 dict 包含 `id`, `title`, `content`, `keywords`, `source`。将 `content` 作为文档，其他作为元数据，使用卡片的 `id` 作为 Chroma 的文档 ID。
- 函数 `search_similar(query: str, k=5) -> list[dict]`：返回与查询最相似的 k 个文档，每个结果包含 `id`, `title`, `content`, `source`，以及 `similarity`（距离或相似度，Chroma 默认返回距离，可转为 1-距离）。
- 函数 `add_golden_sample(event, analysis, rating)`：追加到一个独立的集合 `golden_samples`，方便未来导出。这部分可选。
#### seed_cards.json
- 提供一个有效的 JSON 数组示例，包含 3–5 条理论卡片，格式：
```json
[
  {
    "id": "tragedy_of_commons_01",
    "title": "公地悲剧",
    "content": "当多个个体共同使用一种稀缺资源时，个体理性导致资源枯竭。在Minecraft中常见于公共矿道、农田被过度开采且无人维护。",
    "keywords": ["公共资源", "个体理性", "资源枯竭"],
    "source": "Hardin, G. (1968). The Tragedy of the Commons."
  },
  ...
]
```
llm_api.py
使用 openai 库的 OpenAI 客户端，base_url 设为 config.DEEPSEEK_BASE_URL，api_key 来自环境变量。
函数 generate_analysis(event_text: str, retrieved_cards: list[dict]) -> str：
构建系统提示（system prompt）：让模型扮演 Minecraft 社科研究者，只能依据提供的参考理论，输出格式包含：事件概要、理论分析（分点并引用理论ID）、引申问题。
用户消息包含：检索到的卡片引用格式（编号、标题、内容）+ 事件文本。
调用 client.chat.completions.create，model 设为 deepseek-chat，temperature 0.3 左右保证严谨。
返回模型的回答文本。
函数 generate_analysis_stream(...)：同上但流式输出，用于在 Streamlit 界面实时打字。
utils.py
函数 parse_references(analysis_text: str) -> list[str]：用正则提取分析文本中的引用ID（如 [引用: ...]），返回ID列表。
函数 clean_event_text(text: str) -> str：简单的清洗，去除多余空行。
函数 save_golden_sample(event, analysis, rating)：追加到一个本地 JSONL 文件 golden_samples.jsonl。
app.py（Streamlit 主界面）
页面标题：“Minecraft 游戏社科分析台”
侧边栏
显示当前知识库统计（卡片总数、最近更新日期）
提供“添加理论卡片”表单：id、标题、内容、关键字、来源。提交后调用 knowledge_base.add_cards。
提供“批量导入卡片”按钮：上传 JSON 文件，解析后批量添加。
提供“检索测试”小工具：输入查询词，展示相似卡片及相似度。
主页布局：
上方：一个大的文本区域，让用户输入“游戏事件描述”（结构化事件文本）。
按钮“提交分析”。
分析时显示加载动画，并可切换流式输出。
结果区：显示 AI 分析全文，其中引用部分（如 [引用:xxx]）用彩色高亮，并在右侧或底部展开显示对应理论卡片的原文。
每条分析下方有三颗星评分（1-3），和“保存为黄金样本”按钮。点击后调用 utils.save_golden_sample。
状态维护：使用 st.session_state 保存对话历史或当前分析结果。
5. 关键实现细节
首次运行时，如果 Chroma 集合为空，自动从 seed_cards.json 加载初始卡片。
嵌入模型只加载一次，使用 @st.cache_resource 装饰 embedding 模型和 Chroma 客户端，以便在 Streamlit 中高效复用。
调用 LLM 时，如果 API 不可用，给出友好的错误提示。
整个系统应在 Windows / Linux / macOS 上均可运行，路径使用 os.path.join。
